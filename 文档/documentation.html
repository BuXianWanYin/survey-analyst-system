在线问卷调查与数据分析系统

系统说明文档 - 项目概述、技术栈、架构设计、功能模块、答辩准备

项目概述

项目名称：在线问卷调查与数据分析系统

开发模式：前后端分离

维护者：开发团队

完成时间：2025年

开发进度：已完成

项目简介：本系统是一个基于Web的在线问卷调查与数据分析系统，采用B/S架构，支持问卷设计、发布、数据收集、多维度分析和可视化展示等功能。

快速启动

环境要求

在开始之前，请确保已安装以下软件：

  • JDK 1.8 或更高版本 - Java开发环境
  • Maven 3.6 或更高版本 - Java项目构建工具
  • MySQL 8.0 或更高版本 - 关系型数据库
  • Redis 6.0 或更高版本 - 缓存数据库
  • Node.js >= 16.0.0 - JavaScript运行环境
  • npm >= 7.0.0 - Node.js包管理器

后端启动步骤

步骤1：创建数据库

CREATE DATABASE survey_analyst CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

在MySQL中执行上述SQL语句创建数据库。

步骤2：配置数据库连接

编辑 survey-analyst-server/src/main/resources/application-dev.yml 文件，配置数据库连接信息：

spring: datasource: url: jdbc:mysql://localhost:3306/survey_analyst?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai username: root password: your_password

步骤3：配置Redis连接

在同一配置文件中配置Redis连接信息：

spring: data: redis: host: localhost port: 6379 password: your_redis_password # 如果有密码 database: 0

步骤4：编译项目

cd survey-analyst-server mvn clean install

在项目根目录执行Maven编译命令。

步骤5：运行项目

mvn spring-boot:run

或者直接运行 SurveyAnalystServerApplication.java 的main方法。

步骤6：访问API文档

启动成功后，访问 http://localhost:8080/doc.html 查看API文档。

前端启动步骤

步骤1：安装依赖

cd survey-analyst-web npm install

在项目根目录执行npm安装命令，安装所有依赖包。

步骤2：配置环境变量

在项目根目录创建 .env.development 文件，配置以下内容：

# 应用标题 VITE_APP_TITLE=在线问卷调查与数据分析系统 # API 基础路径 VITE_APP_BASE_API=/api # 请求超时时间（毫秒） VITE_APP_TIMEOUT=30000 # 开发服务器端口 VITE_SERVER_PORT=3001 # 开发服务器主机地址 VITE_SERVER_HOST=0.0.0.0 # 代理目标地址（后端服务地址） VITE_SERVER_PROXY_TARGET=http://localhost:8080

步骤3：启动开发服务器

npm run dev

启动Vite开发服务器，支持热更新。

步骤4：访问前端

启动成功后，访问 http://localhost:3001 查看前端界面。

请求流程

  1. 前端发起请求 - 用户在前端界面进行操作，前端通过Axios发送HTTP请求
  2. API网关路由 - Nginx接收请求，根据路径进行反向代理，转发到后端服务
  3. 后端处理请求 - Spring Boot接收请求，经过安全认证、参数验证等中间件处理
  4. 业务逻辑处理 - Service层执行业务逻辑，调用Mapper层进行数据库操作
  5. 数据存储/缓存 - 数据持久化到MySQL，统计数据缓存到Redis
  6. 返回响应 - 处理结果通过Controller返回JSON格式数据给前端
  7. 前端渲染 - 前端接收数据，更新界面显示

后端结构

后端采用经典的分层架构：

  • Controller层 - 接收HTTP请求，参数验证，返回响应结果
  • Service层 - 业务逻辑处理，事务管理，调用Mapper层
  • Mapper层 - 数据访问层，使用MyBatis Plus进行数据库操作
  • Entity层 - 实体类，对应数据库表结构
  • DTO/VO层 - 数据传输对象和视图对象，用于接口参数和返回值
  • Config层 - 配置类，包括Spring Security、Redis、MyBatis Plus等配置
  • Utils层 - 工具类，提供通用功能方法
  • Aspect层 - 切面类，实现操作日志记录等功能

前端结构

前端采用组件化开发模式：

  • Views层 - 页面组件，包括问卷管理、数据分析、系统管理等页面
  • Components层 - 公共组件，包括表单渲染、问卷预览等可复用组件
  • API层 - API接口封装，统一管理所有后端接口调用
  • Router层 - 路由配置，定义页面路由和路由守卫
  • Store层 - 状态管理，使用Pinia管理全局状态
  • Utils层 - 工具函数，包括请求封装、日期处理、图表配置等
  • Layouts层 - 布局组件，提供统一的页面布局结构

核心业务流程

问卷设计流程

步骤1：创建问卷

用户登录系统后，点击"创建问卷"按钮，填写问卷标题、描述，上传封面图片。系统自动创建问卷并设置为草稿状态。

步骤2：设计问卷内容

使用拖拽式问卷编辑器添加题目，支持多种题型：

  • 单选题、多选题、下拉选择
  • 单行文本、多行文本
  • 数字、日期、时间、日期时间
  • 评分题、滑块题
  • 图片选择、级联选择
  • 文件上传、图片上传
  • 签名题、图片展示、图片轮播
  • 描述文本、分割线

每个题目可以配置属性：必填/非必填、提示文字、验证规则、默认值等。

步骤3：配置逻辑跳转

在逻辑配置页面，可以设置题目之间的跳转关系：根据选项跳转到指定题目、根据条件显示/隐藏题目、分页逻辑控制等。逻辑规则以JSON格式存储。

步骤4：设置外观主题

在主题设置页面，可以自定义问卷的外观：主题颜色、背景颜色、Logo上传与位置、头图设置、按钮样式等。支持移动端和桌面端预览。

步骤5：保存问卷

设计完成后，问卷自动保存为草稿状态，可以随时编辑和修改。

问卷发布流程

步骤1：发布设置

在发布设置页面，配置以下选项：

  • 访问权限 - 公开访问或密码访问
  • 填写限制 - IP限制、设备限制、账号限制、填写次数限制
  • 时间设置 - 开始时间、结束时间
  • 目标填写数量 - 设置期望的填写数量
  • 通知设置 - 填写完成后的通知方式

步骤2：发布问卷

点击"发布"按钮，系统验证问卷配置（表单配置、表单项是否存在），验证通过后将问卷状态变更为"已发布"（PUBLISHED），问卷开始接收填写数据。

步骤3：生成分享链接和二维码

发布成功后，系统自动生成问卷访问链接和二维码图片。用户可以通过链接或扫描二维码访问问卷，支持多渠道分享（微信、QQ、微博等）。

数据收集流程

步骤1：访问问卷

填写者通过分享链接或扫描二维码访问问卷。如果问卷设置了密码访问，需要先输入密码验证。系统记录访问IP、设备类型等信息。

步骤2：填写问卷

填写过程中：

  • 实时表单验证 - 必填项检查、格式验证等
  • 逻辑跳转执行 - 根据选项自动跳转到指定题目
  • 草稿保存（可选） - 可以保存填写进度，稍后继续
  • 填写进度显示 - 显示当前填写进度百分比

步骤3：提交数据

点击提交按钮后：

  • 前端验证 - 检查必填项、格式等
  • 后端验证 - 服务器端再次验证数据有效性
  • 防重复提交检查 - 根据IP、设备、账号等限制检查
  • 数据存储 - 将填写数据存储到数据库（form_data表）

步骤4：记录填写信息

系统自动记录填写信息到response表：IP地址、设备类型（PC/MOBILE）、填写时长（秒）、开始时间、提交时间等，用于后续的数据分析。

数据分析流程

步骤1：数据统计

系统提供实时数据统计功能：

  • 问卷整体统计 - 总填写数、完成数、草稿数、完成率、有效率等
  • 题目统计 - 每个题目的填写数、填写率、选项分布等
  • 实时刷新 - 支持定时刷新（如30秒）获取最新统计数据

步骤2：基础分析

对收集的数据进行基础分析：

  • 选项分布分析 - 使用柱状图、饼图展示各选项的选择人数和比例
  • 文本分析 - 对填空题进行词云分析，提取高频词
  • 评分分析 - 计算平均分、最高分、最低分，展示评分分布

步骤3：高级分析

提供多维度的高级分析功能：

  • 交叉分析 - 分析多个题目之间的关联关系，生成交叉表
  • 趋势分析 - 按时间维度统计填写趋势，分析选项选择率随时间的变化
  • 画像分析 - 分析填写者的特征，包括地域分布、设备偏好、填写时间偏好等
  • 对比分析 - 支持按不同条件筛选数据后进行对比分析

步骤4：数据可视化

使用ECharts实现丰富的数据可视化：

  • 多种图表类型 - 柱状图、饼图、折线图、雷达图、词云图等
  • 图表交互 - 支持缩放、筛选、数据提示等交互功能
  • 数据导出 - 支持导出Excel、PDF、图片等格式

认证方式

系统使用JWT Token进行身份认证：

  • 登录接口（/api/auth/login）无需认证，可直接调用
  • 其他接口需要在请求头中携带 JWT Token
  • Token 格式：Authorization: Bearer {token}
  • Token 有效期：24小时（可在配置文件中修改）

答辩准备

常见问题及回答

1. 系统的核心功能是什么？

系统的核心功能包括：问卷设计（拖拽式编辑器）、问卷发布（多渠道分享）、数据收集（实时提交）、数据统计（实时统计）、数据分析（交叉分析、趋势分析、画像分析）、数据可视化（多种图表展示）、数据导出（Excel、PDF等格式）。

2. 采用了哪些技术栈？为什么选择这些技术？

后端采用Spring Boot 2.6.13，因为Spring Boot简化了配置，提高了开发效率；前端采用Vue 3.4.21，因为Vue 3的Composition API提供了更好的代码组织和复用性；数据库使用MySQL 8.0+，因为MySQL成熟稳定，适合结构化数据存储；使用Redis作为缓存，提高系统性能。

3. 系统的创新点在哪里？

创新点包括：①拖拽式问卷设计器，提供直观的可视化编辑体验；②多维度数据分析，支持交叉分析、趋势分析、画像分析等高级分析功能；③实时数据统计，支持定时刷新获取最新统计数据；④完善的逻辑跳转功能，支持复杂的问卷逻辑；⑤多种数据可视化展示，包括柱状图、饼图、折线图、词云图等。

4. 遇到了哪些技术难点？如何解决的？

主要技术难点：①问卷逻辑跳转的复杂实现 - 通过JSON配置存储逻辑规则，前端解析执行；②大量数据的统计分析性能 - 使用Redis缓存统计数据，定时刷新机制；③文件上传和存储管理 - 采用本地文件系统存储，按日期分类管理；④跨域和认证问题 - 使用JWT Token无状态认证，配置CORS解决跨域。

5. 系统的性能如何？如何优化的？

性能优化措施：①使用Redis缓存统计数据，减少数据库查询；②数据库索引优化，对常用查询字段建立索引；③分页查询，避免一次性加载大量数据；④前端代码分割和懒加载，提高页面加载速度；⑤使用连接池（Druid）管理数据库连接，提高连接复用率。

技术亮点

  • 前后端分离架构 - 提高开发效率，便于团队协作和维护
  • 拖拽式问卷设计器 - 提供直观的可视化编辑体验，降低使用门槛
  • 多维度数据分析 - 支持交叉分析、趋势分析、画像分析等高级分析功能
  • 实时数据统计 - 支持定时刷新获取最新统计数据，实时掌握问卷填写情况
  • 完善的权限管理 - 基于Spring Security和JWT的安全认证机制
  • 多种数据导出格式 - 支持Excel、PDF、CSV等多种格式导出
  • 响应式设计 - 支持PC端和移动端访问，适配不同设备
  • 逻辑跳转功能 - 支持复杂的问卷逻辑，提供灵活的问卷设计能力

难点解决

1. 问卷逻辑跳转实现

问题：如何实现复杂的问卷逻辑跳转？
解决方案：使用JSON格式存储逻辑规则，定义条件表达式和跳转目标，前端解析JSON规则并实时执行跳转逻辑。

2. 大量数据统计分析性能

问题：大量填写数据的统计分析性能如何保证？
解决方案：使用Redis缓存统计数据，设置合理的缓存过期时间，提供手动刷新接口，采用定时刷新机制获取最新数据。

3. 文件上传和存储管理

问题：如何管理上传的文件？
解决方案：采用本地文件系统存储，按日期分类存储（upload/yyyy/MM/dd/），提供文件信息查询和删除接口，确保文件安全访问。

总结与展望

工作总结

本系统成功实现了在线问卷调查与数据分析的核心功能，包括问卷设计、发布、数据收集、统计分析和数据可视化等。系统采用前后端分离架构，使用Spring Boot和Vue 3等现代化技术栈，提供了良好的用户体验和系统性能。通过拖拽式问卷设计器、多维度数据分析、实时统计等功能，系统能够满足用户进行在线问卷调查和数据分析的需求。

系统不足

  • 数据量较大时，统计分析性能仍有优化空间
